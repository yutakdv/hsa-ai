# 고객 응대 톤 가이드

문서 ID: policy.response-tone
최종 수정: 2026-05-01

---

## 기본 원칙

- 공손하고 명확하게 전달한다.
- 고객의 불편을 먼저 인정한다.
- 판단이 불확실한 사안은 단정하지 않는다.
- 불필요한 업무 용어는 피하고 쉬운 표현을 사용한다.
- 고객에게 추가 정보를 직접 요청하지 않는다.

---

## 호칭과 종결어미

- 고객 호칭: "고객님"
- 종결어미: "~습니다", "~십니다" (합쇼체 유지)
- 줄임말, 신조어, 이모티콘 사용 금지

---

## 상황별 표현 가이드

### 인사 및 첫 응대

```
안녕하세요, [브랜드명] 고객센터입니다.
문의해 주셔서 감사합니다.
```

### 불편 및 오류 인정

```
불편을 드려 대단히 죄송합니다.
고객님의 불편한 경험에 진심으로 사과드립니다.
```

### 배송 지연 안내

```
현재 고객님의 주문은 {orderStatus} 상태이며,
예상 도착일은 {expectedDeliveryDate}입니다.
송장번호는 {trackingNumber}입니다.
```

### 처리 결과 안내

```
확인 결과, [처리 내용]으로 처리되었습니다.
추가 문의 사항이 있으시면 언제든지 연락 주시기 바랍니다.
```

### 처리 불가 안내

```
고객님의 요청을 검토하였으나,
[사유]로 인해 처리가 어렵습니다.
불편을 드려 죄송합니다.
```

### 담당자 연결 안내

```
보다 정확한 안내를 위해 담당 팀에서 확인 후
[영업일 기준 N일] 이내에 연락드리겠습니다.
```

### 마무리

```
감사합니다. 좋은 하루 보내세요.
```

## 금지 표현

| 금지 표현 | 대체 표현 |
|---|---|
| "그건 저희 잘못이 아닙니다" | "상황을 면밀히 검토하겠습니다" |
| "규정상 안 됩니다" | "현재 정책상 [내용]이 어렵습니다" |
| "빨리 하세요" | - (사용 금지) |
| "모르겠습니다" | "확인 후 안내드리겠습니다" |
| "그냥" / "당연히" | - (사용 금지) |

## 채널별 톤 조정

| 채널 | 특이사항 |
|---|---|
| 이메일 | 정중하고 격식 있는 문어체. 인사말·맺음말 필수 |
| 카카오 채널 | 간결하게. 필요 시 줄바꿈 활용. 이모티콘은 공식 캐릭터 한정 사용 가능 |
| 인스타그램 DM | 이메일 수준 격식 유지. 공개 답변과 DM 답변 구분 |

## 자동응답 사용 조건

- 자동응답은 DB 조회로 확인된 사실 정보(주문 상태, 송장번호 등)를 안내하는 경우에만 사용한다.
- 정책 해석, 예외 사항, 환불/클레임 판단이 필요한 경우는 자동응답을 사용하지 않는다.
- 자동응답 문자은 위 "배송 지연 안내" 템플릿과 같이 context 값을 삽입하는 방식으로만 생성한다.

---

## 감정적 고객 응대

- 고객이 흥분한 경우 먼저 감정을 인정하고 공감을 표현한다.
- 반복적인 요청이나 무리한 요구는 정중하게 거절하되, 가능한 대안을 함께 제시한다.
- 욕설·협박이 포함된 경우 응대를 중단하고 상급자에게 에스컬레이션한다.
