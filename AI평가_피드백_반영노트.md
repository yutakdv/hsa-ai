# AI 평가 고도화 — 발표 피드백 정리 및 반영 방향

> 2026-06-25 중간 발표 피드백 정리. report.md / AI파트 진행사항보고서 / evals 코드 기준으로 현재 보유 항목과 보완 방향을 매핑함.

## 1. 받은 피드백 요약

오늘 발표 피드백은 한 줄로 묶이지 않는 네 갈래였지만, 정리하면 다음과 같다.

1. **품질 평가의 정량화 부족** — 분류 정확도·자동응답 분기 정확도 같은 합/불 지표는 있으나, "초안 품질" 자체를 세분화해 수치로 보여주는 단계까지는 가지 못함.
2. **리스크 관점의 부재** — 어떤 자동화 지점이 실제로 위험한 결정인지, 틀렸을 때 비용이 얼마나 다른지가 구조적으로 드러나지 않음.
3. **사용자(운영자) 가치 검증 부족** — "AI가 분류·초안을 만들었다"는 사실은 보여주지만, 그게 운영자의 실제 업무 부담을 얼마나 줄였는지는 아직 수치로 없음.
4. **스코프 적정성에 대한 자기 점검 필요** — Reranker, threshold 환경변수화처럼 정교하게 다듬은 부분이 PoC 규모에 비해 과투자된 것은 아닌지 점검 필요.

## 2. 각 피드백에 대한 반영 방향

### (1) 정량적·품질 평가 고도화

**현재 보유**
- 공통 게이트: 분류 정확도 ≥80%, 자동응답 분기 정확도 ≥85%, Pydantic 검증 통과율 ≥95%, RAG 근거 일치율 ≥80%
- RAG 전용 게이트: source 포함률 ≥80%, 무관 문의 거절률 ≥80%, prompt injection 누출 0건, p95 latency 기준

**보완 방향**
- PASS/FAIL 이진 판정 위주인 채점을, 초안의 정확성·어조·완성도 등을 축으로 나눈 다단계 rubric으로 확장
- 이미 "남은 작업"으로 명시돼 있던 **LLM self-reported confidence calibration**(보고된 confidence와 실제 정답률의 상관관계 측정)을 우선순위로 끌어올려, "이 시스템이 확신할 때 실제로 맞는가"를 별도 지표로 제시
- 자동응답/RAG 분기 오류를 confusion matrix로 시각화해 "어떤 오분류가 가장 위험한지"를 한눈에 드러냄

### (2) Risk 관점 정리

**현재 보유**
- `riskTags`(refund, claim, policy_conflict)로 운영자 검토 필요 여부 표시
- 정책 해석·예외 키워드 포함 시 자동응답 자체를 차단하는 구조

**보완 방향**
- 태그를 등급(Low/Medium/High)으로 구체화하고, 등급별로 "AI가 틀렸을 때 발생하는 비용"을 명시 (예: 배송 조회 오답 → 단순 재문의 vs 환불 오답 → 금전적·신뢰 손실)
- 자동응답 영역에서 **false positive(잘못 자동응답된 케이스) = 0건**을 별도 게이트로 명문화 — 현재 자동응답 분기 정확도 85% 안에 묻혀 있는 risky한 오류 유형을 분리해서 추적

### (3) 사용자(운영자) 가치 검증

**현재 보유**
- 처리 기록 대시보드에서 AI 처리/운영자 처리 건수를 일별로 집계 (예: 오늘 처리 28건 중 AI 18건·운영자 10건)

**보완 방향**
- "AI 초안이 없었다면 운영자가 직접 작성하는 데 걸렸을 시간" vs "초안을 검토만 하는 데 걸린 시간"을 비교해 실제 절감 효과를 수치로 제시
- 단순 처리 건수보다 **초안을 수정 없이 그대로 발송한 비율** 같은, 초안의 실사용성을 보여주는 지표를 추가 — 이게 결국 "사용자에게 도움이 되는가"에 대한 가장 직접적인 답이 됨

### (4) 스코프 적정성 자기 점검

- Reranker·primary-policy 필터·threshold 환경변수화는 효과가 이미 측정되어 있음(대표 source/거절 동작 6/6 PASS, rerank 도입으로 응답시간 34% 개선) → "부풀려진 개발"이 아니라 "측정 가능한 개선"이라는 근거를 포트폴리오에 더 명확히 제시
- 반면 VectorDB(Chroma/Qdrant/pgvector) 도입처럼 아직 효과가 없는 항목은 "문서 수가 실제로 늘어났을 때"라는 트리거 조건을 달아 PoC 단계에서는 우선순위를 낮춤을 분명히 함 — 즉 모든 고도화가 아니라 **측정된 것만 했다는 기준**으로 스코프를 정당화
